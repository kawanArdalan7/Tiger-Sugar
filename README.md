## Project Overview

Tiger Sugar is a full-stack web application designed to modernize and streamline the ordering and management workflow of a popular boba café. Built with **React**, **Node.js**, and **PostgreSQL**, the platform supports **three distinct user roles**—customers, cashiers, and managers—each with tailored interfaces and functionality.

The application prioritizes **accessibility**, **usability**, and **operational efficiency**, featuring:

- Real-time inventory and order tracking
- Multi-language support
- Scalable UI for visual accessibility
- Weather-aware UI elements
- Secure authentication for staff

This project was developed by a **team of four** using Agile methodologies as part of a university capstone-style course.

> _Note: This project is a **non-commercial, academic recreation** of **[Tiger Sugar's][tiger-sugar-original-link]** existing system._

## Problem Statement

Traditional café POS and ordering systems often struggle to:

- Serve users with accessibility or language needs
- Support multiple user roles cleanly
- Provide real-time operational insights to management

Our goal was to design a system that adapts to **different user needs** while remaining intuitive, fast, and inclusive.

<p align="right">(<a href="#readme-top">back to top</a>)</p>

---

## Proposed Solution

We developed a role-based web platform that:

- Separates customer, cashier, and manager workflows
- Reduces friction in ordering and checkout
- Improves visibility into inventory, sales, and restocking
- Enhances accessibility through UI scaling and translation

The result is a system that improves **speed**, **accuracy**, and **user satisfaction** across all roles.

## Project Roadmap

### Pre-Development (Oct 16 - Oct 24)

#### User Research

Focus on accessibility, clarity, and ease of use--especially for:

- Visually impaired users
- Non-English speakers

Key findings:

- Need for larger, scalable text
- Clear navigation hierarchy
- Improved cashier notifications
- More inviting visual design

#### Expert Review

Feedback emphasized:

- Strong separation of user roles
- Accessibility-first design
- Language and scaling support

Actions taken:

- UI/UX redesign
- Role-specific interfaces
- Accessibility tooling integration

---

### Sprint 1 - Core Foundations (Oct 30 - Nov 7)

- [x] Database connection and menu retrieval
- [x] Static customer menu view
- [x] Navigation bar with accessibility options
- [x] Staff login entry point
- [x] Staff-specific navigation
- [x] Basic cart functionality (add/remove/checkout)
- [x] Payment flow (cash or card)
- [x] Inventory data visibility

---

### Sprint 2 - Feature Expansion (Nov 13 - Nov 21)

- [x] Employee database cleanup
- [x] Menu database restructuring
- [x] Menu item images and categories
- [x] Interactive item customization pop-ups
- [x] Order history with date grouping
- [x] Inventory management (add/update/delete)
- [x] Menu management (add/update/delete)
- [x] Staff logout handling

---

### Sprint 3 - Polish & Advanced Features (Nov 27 - Dec 5)

- [x] UI scaling controls
- [x] Google Translate integration
- [x] OAuth-based staff authentication
- [x] Weather API integration and display
- [x] Dynamic menu rendering from database
- [x] Visual cart and checkout indicators
- [x] Order completion feedback
- [x] Manager reports:
  - [x] Restock report
  - [x] Excess inventory report
  - [x] Sales report

I served as the **Frontend Lead** on this project, with ownership over the customer-facing experiences, accessibility features, and frontend architecture. My responsibilities included:

- Leading frontend design and implementation using React and MUI
- Designing and building customer ordering flows end-to-end
- Implementing accessibility features (UI scaling, contrast, assistive tooling)
- Integrating third-party APIs (Google Translate, weather services)
